# CHANGELOG Policy

Политика обязательна к применению в upstream ARIA. Применяется механически через `scripts/hooks/pre-commit` и семантически через ссылку в `core/CLAUDE.md.template`.

---

## Область действия

**Для upstream ARIA (обязательно):**
- Любое изменение в `core/`, `adapters/`, `docs/policies/`, `docs/ARIA_GUIDE.md` **ТРЕБУЕТ** запись в `CHANGELOG.md` того же коммита.
- Единственное исключение: коммит, который изменяет **только** `CHANGELOG.md` (типо-фикс в самом файле).

**Для форков (рекомендовано, но не обязательно):**
- У форка свой `CHANGELOG.md` для задач проекта (тот же гибридный формат — scope-секции + табличные строки).
- Системные изменения (sync с ARIA, изменение команд форка) идут в `SYS_CHANGELOG.md`.

---

## Гибридный формат CHANGELOG (обратное заимствование из форков, ARIA Phase 1 Task 12, R2+R5)

**Обоснование:** ARIA исходный формат был scope-секциями по версиям (агрегация релиза). Форки использовали табличный формат с детальной прослеживаемостью на уровне задачи (дата/task/SHA/файлы/тесты). Гибридный формат совмещает преимущества обоих.

### Структура

```markdown
## v{X.Y.Z} ({YYYY-MM-DD}) — {Краткий заголовок версии}

### Core
| дата | task | SHA | файлы | тесты | атрибуция |
|------|------|-----|-------|-------|-----------|
| 2026-04-14 | fix_X | abc1234 | core/commands/spec.md, core/templates/SPEC.md.template | 42/42 | — |
| 2026-04-13 | feature_Y | def5678 | core/commands/aria-triage.md | 30/30 | from {Fork}, PR #42 |

### Adapters
| дата | task | SHA | файлы | тесты | атрибуция |
|------|------|-----|-------|-------|-----------|

### Docs
| ... |

### Breaking
| ... |

### Fix
| ... |
```

### Колонки таблицы

- **дата** — `YYYY-MM-DD`, локальная дата коммита
- **task** — `task_name` из STATE.yaml или краткое описание
- **SHA** — короткий хэш коммита (git rev-parse --short)
- **файлы** — список через запятую, ключевые изменённые файлы (не все — топ-5 самых значимых)
- **тесты** — `N/M` (прошло/всего) или `—` если неприменимо (например, docs-only коммит или triage-merge без локальных тестов)
- **атрибуция** — `from {fork}, PR #{N}` если принято через `/aria-triage`, иначе `—`

### Секции

- `Core` — изменения в `core/` (команды, шаблоны, протоколы, CLAUDE.md.template)
- `Adapters` — изменения в `adapters/`
- `Docs` — изменения в `docs/`, политиках, ARIA_GUIDE, README
- `Hooks` — изменения в `scripts/hooks/`
- `Breaking` — ломающие изменения (требуют MAJOR версии)
- `Fix` — багфиксы

Секция присутствует если в ней есть хотя бы одна строка. Пустые секции не пишутся.

## Преимущества гибрида

| Что даёт | Scope-секции (ARIA исходное) | Табличные строки (форки) | Гибрид |
|----------|------------------------------|------------------------------|--------|
| Версионирование | ✓ | — | ✓ |
| Агрегация релиза | ✓ | — | ✓ |
| Детальная прослеживаемость задачи | — | ✓ | ✓ |
| Список файлов по задаче | — | ✓ | ✓ |
| Счёт тестов | — | ✓ | ✓ |
| Обратный поиск коммит→задача | — | ✓ (`grep SHA`) | ✓ |
| Поиск атрибуции по форку | — | — (нет колонки) | ✓ (`grep "from {Fork}"`) |

Это **заменяет** CONTRIBUTIONS.md (удалён в Task 12): срез контрибуций = `grep -E 'from [A-Za-z]+, PR' CHANGELOG.md`.

---

## Правило записи даты

ISO 8601: `YYYY-MM-DD`. Локальная дата коммита, не UTC.

---

## Правило версионирования

Semantic versioning:
- **MAJOR (X.0.0)** — ломающие изменения протоколов, форматов документов, удаление артефактов
- **MINOR (X.Y.0)** — новые команды, новые шаблоны, новые адаптеры, новые политики
- **PATCH (X.Y.Z)** — bugfixes, уточнения в документации, правки hooks без изменения контракта

Промежуточные версии в разработке: `v{X.Y.Z}-dev` — `/aria-triage --accept` пишет в текущую `-dev`, `/aria-release` преобразует её в релизную при выпуске.

## Что попадает в CHANGELOG

| Тип изменения | В CHANGELOG? | Секция |
|---------------|--------------|--------|
| Новая команда в `core/commands/` | **Да** | Core |
| Изменение существующей команды (семантика) | **Да** | Core |
| Опечатка в команде | **Да** | Fix |
| Новый адаптер | **Да** | Adapters |
| Изменение `core/templates/*.template` | **Да** | Core |
| Изменение `core/protocols/*.md` | **Да** | Core (часто Breaking) |
| Новая политика в `docs/policies/` | **Да** | Docs |
| Правка `docs/ARIA_GUIDE.md`, README.md, SPEC.md, STACK.md | **Да** | Docs |
| Правка `scripts/hooks/` | **Да** | Hooks |
| Правка `.gitignore`, `.github/` | **Да** | Docs (если PR template / labels), Hooks (если workflows) |
| Правка `scripts/validate_*.sh` | **Да** | Hooks |
| Принят PR от форка (`/aria-triage --accept`) | **Да** | по scope PR + колонка атрибуции `from {Fork}, PR #{N}` |
| Опечатка в CHANGELOG | Нет (пропуск проверки hook) |
| Правка `README.md` без изменения функциональности | Нет | — |
| Правка `TRIAGE.md`, `FORKS.md`, `STATE.yaml`, `ROADMAP.md` | Нет (Two-Phase Commit пишет сам) |

## Механическая валидация (hooks)

`scripts/hooks/pre-commit` проверяет:
1. Если в коммите изменены файлы из `core/`, `adapters/`, `docs/policies/`, `docs/ARIA_GUIDE.md`, `docs/SPEC.md`, `docs/STACK.md` — то `CHANGELOG.md` **тоже** должен быть изменён в этом коммите.
2. Если CHANGELOG.md изменён — в нём должна быть запись:
   - новая строка в существующей `v{X.Y.Z}-dev` секции ИЛИ
   - новая `## v{X.Y.Z}-dev` секция для начала новой версии
3. Строка таблицы должна иметь все 6 колонок (дата/task/SHA/файлы/тесты/атрибуция).

Обход валидации: `git commit --no-verify` (только для экстренных случаев, с обоснованием в сообщении коммита).

---

## Семантическая валидация (CLAUDE.md)

`core/CLAUDE.md.template` содержит обязательную ссылку на эту политику. Любой AI-агент, работающий в ARIA, обязан читать политику при подготовке коммита.

---

## История политики

- **v1.0 (исходная):** scope-секции без табличной детализации. Потеря трассировки файлов/тестов.
- **v2.0 (2026-04-15):** гибридный формат (scope + таблицы). Обратное заимствование из форков. Закрывает регрессии R2 и R5 Phase 1.
- **v2.0:** CONTRIBUTIONS.md удалён — атрибуция мигрировала в колонку таблицы CHANGELOG.

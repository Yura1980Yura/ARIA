3. Примени исправления
4. МАШИННАЯ ПРОВЕРКА E2E Testing Plan (ОБЯЗАТЕЛЬНО):
   ```bash
   bash "${{ENV_CODE}}/scripts/validate_spec_e2e.sh" {specs}/{task_name}_draft.md
   ```
   Скрипт проверяет 13 пунктов:
   □ Секция "## E2E Testing Plan" существует
   □ Затронутые механики заполнены (≥1 файл→механика)
   □ Есть ID тестов (E2E-/API-) — новые и/или регрессионные
   □ {{SMOKE_TEST_ID}} включён (обязательный smoke)
   □ AC привязаны к тестам (ПОЛНАЯ проверка: каждый AC → Test ID)
   □ Антипаттерны задачи ≥ 2
   □ Метрики заполнены числами (min_new_tests, min_regression_tests, min_total_checks)
   □ TCM существует (секция "### Test Coverage Matrix")
   □ TCM полная (все 7 подсекций имеют минимум 1 строку)
   □ Каждая строка TCM имеет "Тест?" = да
   □ GDQ секция существует (10 вопросов)
   □ GDQ все 10 вопросов имеют ответы (не пустые)
   □ Новые тесты ≥ min_new_tests из метрик

   SPEC_E2E_PASS → продолжай.
   SPEC_E2E_FAIL → ВЕРНИСЬ и доработай. Не записывай финальный файл.

# ARIA — Triage (входящие предложения от форков)

> **Обрабатывается командой `/aria-triage`** (upstream-maintainer).
> **Писатель:** `/aria-triage` — append-запись активного PR, `--accept/--decline/--discuss` перемещает в архив.
> **Читатель:** `/aria-triage` при следующем запуске (определение уже зарегистрированных PR).
> Пользователь не редактирует этот файл вручную.

**Источник PR:** `gh pr list --label contribute-back --state open`
**Формат атрибуции при принятии:** `| from {fork}, PR #{N} |` в CHANGELOG.md табличной строки.

---

## Активные предложения

| ID | Дата | Форк | PR/Issue | Scope | Описание | Статус |
|----|------|------|----------|-------|----------|--------|

*(Пусто — ещё не было входящих contribute-back PR. Первые записи появятся после запуска `/aria-triage`.)*

**Возможные статусы:**
- `pending` — PR собран `/aria-triage`, ждёт анализа или решения
- `analyzed` — отчёт сформирован, ждёт одобрения пользователя
- `discussing` — добавлен комментарий в PR с запросом деталей (через `/aria-triage --discuss`)
- `accepting` — принято, идёт merge + обновление CHANGELOG

|----|-------------|------|----------|---------|-------------|---------|

*(Пусто — архив заполняется по мере обработки активных предложений.)*

**Возможные решения:**
- `accepted` — смерджено, атрибуция в CHANGELOG v{X.Y.Z}
- `declined` — закрыт с комментарием
- `superseded` — вытеснено более поздним PR на ту же тему
